1. 사용할 외부 API (무료·공개 위주 관점)

시드는 실제 악성 실행·실제 유해 URL·실제 파일 시스템 접근 없음을 전제로 하므로, “보안 스캔/평판”류 기능은 외부 상용·위협 인텔 API로 대체하지 않고 모의 도구로 구현하는 것이 요구사항과 일치합니다.

구분 API / 서비스 용도 비고
필수 OpenAI API 대화·추론·ReAct 에이전트 (모델 gpt-5-mini 고정) 과금 서비스이나 본 프로젝트의 핵심 외부 연동. temperature, top_p 등 gpt-5 계열 미지원 파라미터 사용 금지.
핵심 설계 (없음 — 모의 응답) USB 스캔, 파일 평판, 설치 미리보기, 크랙 유혹 페이지 등 무료 외부 API로 실제 위협 데이터를 끌어오면 안전·법적·교육 목적과 충돌할 수 있어 시드상 모의가 원칙.
선택 (무료·공개) Wikipedia / MediaWiki API 용어 설명·배경 지식 문구 보강(예: 멀웨어, 피싱 일반론) 시나리오 팩트는 여전히 가짜 자산 유지. 네트워크·레이트 제한 고려.
선택 (무료·공개) REST Countries 등 순수 정보성 공개 API UI 데모·무관한 스타일 참고용 보안 랩 본질과 무관; 과제 범위 밖이면 생략 가능.
정리: “무료 API 위주”는 도구 결과를 만드는 측면에서는 모의 함수가 본체이고, 외부 호출이 필요하다면 OpenAI(필수)와, 교육 문구 보강용으로만 제한적인 공개 API를 두는 구성이 시드와 가장 잘 맞습니다.

2. 실제 구현할 Tool 리스트

시드 mock_tools_suggested를 구현 단위로 구체화했습니다. 이름·역할은 시드와 동일 계열로 맞추면 됩니다.

Tool 이름 (제안) 목적 입력·출력(개략)
mock_usb_device_scan 가짜 USB 라벨·용량·자동실행 힌트 반환 예: 가상 드라이브 레터 또는 시나리오 id → 모의 메타데이터·경고 힌트 JSON/문자열
mock_file_reputation 가짜 해시·출처·커뮤니티 반응 요약 예: 가상 파일명/해시 → 모의 평판 점수·코멘트 요약
mock_installer_preview 가짜 설치 마법사 단계·의심 행위 플래그 예: 가상 설치 패키지 id → 단계 목록·플래그(예: 추가 SW 체크박스 등)
mock_crack_offer_page 가짜 크랙 사이트 스니펫·위험 신호(설문/교육용) 예: 시나리오 분기 코드 → 모의 HTML 스니펫·위험 신호 목록

그래프·앱 레벨 (Tool이 아닐 수 있음)

  • 미션 전환 / 완료 판정: current_mission, completed_missions, mission_phase를 갱신하는 노드 또는 헬퍼 (수락 기준: 순차 미션 및 완료 시 다음 미션).
  • 분기: branch_flags 및 도구 결과에 따른 조건부 엣지 (수락 기준: 최소 한 곳에서 도구 결과에 따른 그래프 분기).
  • 추적: tool_trace에 호출·결과 요약 적재 → Streamlit 패널에 표시 (확장 가능한 형태 허용).

3. LangGraph 워크플로 시각화

아래 다이어그램은 시드의 하이브리드 아키텍처를 반영합니다. 상위는 StateGraph로 미션·분기를 명시하고, 각 미션 내부는 ReAct(create_react_agent 등)로 대화·도구 호출을 처리합니다. 브라우저에서 Mermaid가 렌더되지 않으면 mermaid.live에 코드를 붙여넣어 확인할 수 있습니다.

3.1 상위 미션 파이프라인 (순서 및 종료)

4. 그 외 요구사항 정리

4.1 제약·비목표

  • 구현 위치: ouro_p2/ (기존 streamlit_react_app.py와 분리 가능).
  • 의존성: 상위 requirements.txt와 호환 (langgraph, langchain-openai, langchain-core, streamlit, python-dotenv 등).
  • 안전: 가짜 자산만 사용; 실제 악성 실행·실제 유해 URL·실제 개인정보 수집 없음; 사용자 실제 파일 시스템 읽기/쓰기 없음.
  • UI 고지: 첫 화면·푸터 등에 교육용 시뮬레이션 명시.
  • 비목표: 실제 멀웨어 실행/배포, 실제 크랙·불법 SW 링크/파일, 자격증명 저장, 불법 침해 교육.

4.2 UI

  • 프레임워크: Streamlit.
  • 구성: 미션 진행 패널(현재 미션·단계·분기 요약), 채팅 영역.
  • 톤: 몰입형이되 항상 가상 시뮬레이션임이 드러나야 함.

4.3 수락 기준 (체크리스트)

  • 세 미션 usb → download → crack 순서 진행; 완료 조건 충족 시 다음 미션 전환.
  • 최소 한 곳에서 도구 호출 결과에 따른 그래프 분기.
  • 각 미션에서 ReAct 에이전트가 하나 이상 도구 호출 가능; Streamlit에서 도구 사용 추적 가능(확장 가능 형태).
  • 세 미션 완료 후 최종 요약(학습 포인트) 표시.
  • 앱 구동 시 및 주요 화면에 교육용 시뮬레이션 고지 표시.

4.4 종료 조건

  • 세 미션 완료 + 최종 요약 생성 → current_mission = completed.
  • 사용자가 체험 종료/리셋 선택 시 안전하게 초기 상태 복귀.
